## 【재무관리 실무 에이전트】

> **관점**: 재무관리부(ALM/유동성관리) 팀장 — mkt-desk 리포트를 ALM 의사결정에 활용

### 1. ALM 핵심 데이터 부재

현재 `bankBond3y` 단일 구간만으로는 조달 커브 구성 불가. 필요 데이터:
- 은행채 1Y/2Y/5Y 발행금리
- 통안채 91일물·182일물·1Y
- KORIBOR 3M/6M (내부 대출 기준금리)
- MMF 설정잔액·머니마켓 수급 동향
- NSFR·LCR 관련 HQLA 가격 추이, 잔존 만기별 현금흐름 갭

### 2. 금리 리스크 계량 분석 미흡

리포트에 금리 수준은 있으나 포지션 영향 분석 전무. ALM 위원회 제출 자료로 사용 불가.

| 필요 분석 | 설명 |
|---|---|
| BPV | 금리 1bp 변동 시 포트폴리오 손익, KTB/은행채/IRS 섹터별 분해 |
| 듀레이션 갭 | 자산 듀레이션 - 부채 듀레이션 추이 차트 |
| NII 시나리오 | ±100bp/+200bp 금리 충격 시 순이자이익 변동 시뮬레이션 |

### 3. 조달 비용 연동 부재

- 은행채 신규 발행금리 vs. 내부이전금리(FTP) 스프레드
- 테너별 조달 비용 곡선 (1M/3M/6M/1Y/2Y/3Y)
- 향후 4주 은행채 만기 도래·신규 발행 예정 캘린더

### 4. 리포트 주기 및 감사 이력

- 현재 daily/weekly/monthly는 ALM 위원회 사이클(월 2회)과 불일치 → ALM위원회 전용 템플릿 필요
- `SavedReport` 구조에 버전·배포 이력·열람자 로그 없음
- 금감원 검사 소명을 위해 필수:
  - 데이터 스냅샷 해시값 보관
  - 배포 이메일 수신자 로그
  - 최소 5년 이력 조회 기능

## 【내부통제 실무 에이전트】

> **관점**: 준법감시부(Compliance) 및 내부통제 담당자
> **결론 선제 제시: 현 구조는 즉각적인 도입이 불가합니다.**

### 1. 법적 위험

| 법령 | 위반 소지 |
|---|---|
| 자본시장법 제174조 | 미공개중요정보 외부(Anthropic) 유출 |
| 개인정보보호법 제17조 | 내부 정보 제3자 제공 |
| 금융지주회사법 | 계열사 정보공유 규정 위반 가능성 |

Anthropic DPA 체결 여부 불명확. 법무 검토 선행 필수.

### 2. 정보보안·데이터 주권

- 로컬 JSON + 별도 인증 없는 접근 → 금융보안원 「금융회사 정보보호 모범규준」 위반
- 외부 LLM 학습 활용 시 데이터 주권 상실 위험
- Anthropic 데이터 보존·활용 정책 법무 검토 필수

### 3. AI 콘텐츠 책임 귀속 불명확

발행 주체(서명자) / AI 검토자 / 최종 승인자 3단계 명확히 분리 필요.
"AI 생성 초안, 담당자 검토·확인 완료" 문구 의무 표기.

### 4. 필수 내부 승인 절차 (현재 전무)

1. IT 아키텍처 심의위원회
2. 개인정보 영향평가(PIA)
3. 정보보안팀 보안성 검토
4. 준법감시인 사전 승인
5. 이사회 보고

### 5. 감사 추적(Audit Trail) 전무

생성·수정·발송 전 단계 로그 없음. 금감원 검사 시 해명 불가.
불변 로그(Immutable Log) 시스템 구축 필수.

> **준법감시부 결론**: 요건 미충족 상태에서의 운영에 대해 **즉시 중단 권고**

## 【IT 실무 에이전트】

> **관점**: IT본부 디지털혁신팀 개발자 — 은행 내부 시스템 도입·운영 실무
> **결론 선제 제시: 현재 상태로는 은행 내부 시스템 도입 사실상 불가**

### 1. 망분리 장벽 — 도입 자체가 막힘

`server.mjs`에서 4개 외부 인터넷 구간 실시간 호출:

| API | 도메인 |
|---|---|
| ECOS | ecos.bok.or.kr |
| FRED | api.stlouisfed.org |
| Yahoo Finance | query1.finance.yahoo.com |
| Anthropic | api.anthropic.com |

업무망에서 전부 차단됨. DMZ 경유 프록시 또는 API 게이트웨이 별도 구축 + Anthropic API 외부 AI 서비스 연동 심의 필요.

### 2. 에러 핸들링·로깅 부재 — 운영 불가

`fetchEcosRows`, `fetchFredValue`, `fetchYahooValue` 모두 `await fetch()` 실패 시 try-catch 없음. 외부 API 타임아웃 한 번에 서버 전체 500 에러. 구조화 로그(Winston/Pino) 전무 → 장애 원인 추적 불가.

### 3. API 키 보안 — 즉시 지적사항

`.env` 평문 저장. 서버 접근 권한만 있으면 키 탈취 가능.
→ HashiCorp Vault / AWS Secrets Manager / 내부 KMS 연동 필요.

### 4. 단일 파일 762줄 — 확장성 한계

라우팅·비즈니스 로직·외부 API 호출·파일 I/O 혼재. 동시 리포트 생성 요청 시 Claude Streaming 처리 중 블로킹 발생.
→ Router/Service/Repository 레이어 분리 + PM2 클러스터링 또는 컨테이너화 필요.

### 5. IT 심의 통과 최소 준비사항

- git 이력 없음 → 소스 형상관리 체계 수립
- CI/CD 파이프라인 없음 → 배포 절차서 + 롤백 계획
- `npm audit` 취약점 스캔 결과 첨부
- 개인정보 처리 여부 개보법 검토
