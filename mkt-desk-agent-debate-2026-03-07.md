## 【자금시장 딜러 에이전트】

> **관점**: 국내 주요 시중은행 자금시장부 수석 딜러 (20년 경력)

### 1. 리포트가 절실한 5가지 실전 상황

- **금통위 당일 오전 7시**: 결정 전 포지션 정리 시간, 30분 안에 금리 레벨·스프레드·NDF 동향 요약 브리핑 필수
- **외국인 국채 대규모 순매도 시**: 10년물 금리 장중 15bp 급등, 리스크관리부가 즉시 익스포저 파악하려면 통합 보고서 필수
- **Fed FOMC 성명 발표 직후 새벽 3시**: 익일 개장 전 헤드라인 임팩트 정리해 이사회에 선제 보고
- **분기 말 LCR 마감**: 재무팀·ALM팀이 시장 조달금리와 내부 자금배분 기준을 동시 확인
- **원/달러 1,450원 돌파 긴급 상황**: 외환팀·자금팀·경영진이 동시에 같은 맥락 공유 필요

### 2. 절약 시간

수작업 90~120분 → 5~10분 단축. 연간 **467시간** 절감 추정.

### 3. 현재 고통 포인트

새벽 FOMC 후 스마트폰 CNBC + Bloomberg + ECOS 복붙 작업. 숫자 오타 한 번에 임원 보고서 오류, 책임은 딜러 몫. 주간 리포트는 금요일 오후 4시 마감인데 실제론 3시부터 자리 못 뜨고 수작업 집계.

### 4. 수신자 니즈

| 수신자 | 니즈 |
|---|---|
| 경영진(부행장·CFO) | "우리 포지션에 뭐가 문제인지" 한 줄 결론 |
| 리스크관리부 | 스프레드 이상 징후·한도 초과 즉시 플래그 |
| 영업점 PB·기업금융팀 | 고객 상담용 시장 현황 요약 |

> **핵심 주장**: 정보 지연이 곧 손실인 시장에서 골든타임을 확보하는 인프라

## 【리스크 관리 에이전트】

> **관점**: 국내 주요 시중은행 리스크관리본부 수석 리스크 매니저, 금감원·BIS 바젤 규정 전문가

### 1. AI 환각(Hallucination) 리스크 — 최고 위험

"한국은행 기준금리 동결" 문구가 실제 인상 결정 직후 자동 발송되면 채권 포지션 오판으로 수백억 원 손실 가능.
→ 모든 AI 생성 문장에 "AI 초안 — 인간 검토 필수" 워터마크 + 담당자 최종 승인 게이트 의무화

### 2. 데이터 정확성 리스크 — 높음

ECOS·FRED API 지연·결측 시 전일 데이터가 당일 수치로 오인.
→ 데이터 타임스탬프 명시, 이상값 알림, Fallback 이중화 필수

### 3. 정보보안 리스크 — 심각

내부 시장 포지션·트레이딩 전략이 Anthropic 외부 API로 전송됨. 금융지주회사법 정보 차단벽 위반, 자본시장법 제174조 미공개중요정보 이용 소지.
→ 프롬프트 익명화, 온프레미스 LLM 전환 또는 Anthropic DPA 체결 선행 필수

### 4. 컴플라이언스 리스크 — 높음

이메일 외부 발송 시 자본시장법상 투자권유·투자분석자료로 간주될 수 있음. 금융투자분석사 서명 없는 시장 전망 배포는 규정 위반.
→ 수신자를 내부 직원으로 제한, 법적 Disclaimer 필수 삽입

### 5. 운영 리스크 — 중간

Claude API·ECOS·FRED 동시 장애 시 조간 브리핑 공백 발생.
→ 장애 감지 모니터링 + 수동 작성 SOP 수립

### 필수 개선 요건 요약

- 인간 최종 승인 워크플로 구현
- 데이터 타임스탬프 및 이상값 검증 로직
- Anthropic DPA 계약 및 프롬프트 익명화
- 내부 전용 배포 제한 및 법적 Disclaimer
- 장애 대응 Fallback SOP 수립

## 【리포트 가치 에이전트】

> **관점**: 국내 증권사 리서치센터 출신, 자산운용사 채권운용 본부장 — 수십 개 금융 리포트 매일 수신·평가

### 1. 실질적 가치의 필수 조건

| 조건 | 내용 |
|---|---|
| 타이밍 | 장 시작 전 7~8시 배포 원칙. 늦은 리포트는 가치 없음 |
| 수치의 맥락화 | 숫자 나열이 아닌 "왜 움직였는가" + "다음 방향은" |
| 포지션 함의 | 읽고 즉시 매매 판단에 연결 가능해야 함 |

### 2. AI 자동생성의 근본적 한계

딜러의 체감 정보 부재. 외국인 국채 선물 순매수 데이터는 있어도 "어느 창구에서, 누가, 헤지인지 투기인지"는 AI 불가. 스왑포인트 뒤의 특정 은행 달러 조달 수요, 크레딧 스프레드의 발행 캘린더 효과 — 이런 **비공개·반공개 정보**가 핵심 인사이트인데 현재 mkt-desk는 이를 전혀 반영하지 못한다.

### 3. 의사결정 연결을 위한 필수 추가 요소

- "금일 3년 IRS pay 관심"처럼 **트레이딩 임플리케이션** 단락 필요
- **리스크 시나리오** ("만약 금통위가 비둘기파적이면") 병기 필수

### 4. 좋은 vs. 나쁜 리포트

| 유형 | 예시 |
|---|---|
| 나쁜 리포트 | "국고 3년 3.45%, 전일 대비 2bp 하락. 외국인 매수 지속." — Bloomberg 화면 복사에 불과 |
| 좋은 리포트 | "3년 구간 강세는 외인 선물보다 국내 보험사 현물 수요 주도. 3/5년 스프레드 축소 중, curve flattener 유효." |

### 5. mkt-desk 개선 제안

- 발행 캘린더, 기관별 포지션 추정 모듈, 전일 장중 특이 호가 패턴 연동
- **상대가치(RV) 관점** 내러티브 필수 포함
- 리포트 말미 "금일 주목 지표·이벤트 체크리스트" 고정 배치

## 【기술 혁신 에이전트】

> **관점**: 핀테크 스타트업 + 대형 금융기관 IT 부서 경험 시니어 소프트웨어 아키텍트

### 1. 핵심 기술 부채

| 문제 | 증상 | 즉각 해결책 |
|---|---|---|
| JSON 파일 저장 | 동시성 충돌·검색 불가·백업 부재 | PostgreSQL 전환 |
| 단일 Express 서버 | AI API 지연(3~10초)이 전체 요청 블로킹 | BullMQ 비동기 작업 큐 도입 |

### 2. 데이터 파이프라인 고도화

- Bloomberg/Refinitiv 추가, node-cron 장중 자동 수집
- Redis 캐시(TTL 5분)로 API 비용·지연 동시 절감

### 3. AI 고도화

- Claude extended thinking으로 금리 방향 근거 정형화
- 과거 리포트 RAG(pgvector) 비교 분석: "이번 주 vs 전월 동기" 자동 대조
- 이상 탐지: Z-score 서버 계산 후 AI 컨텍스트로 전달 (비용 효율적)

### 4. 멀티유저·컴플라이언스

- JWT + Role(analyst/reviewer/admin) 3단계 권한 + 감사 로그 테이블
- Anthropic API → AWS Bedrock(Claude) 또는 Ollama 전환 가능한 **어댑터 패턴** 지금부터 적용
- AES-256 전송 암호화 + DB 컬럼 레벨 암호화

### 5. 우선순위 로드맵

| 단계 | 기간 | 내용 |
|---|---|---|
| 1단계 | 1개월 | PostgreSQL 전환 + 비동기 작업 큐 |
| 2단계 | 2개월 | Redis 캐시 + RAG 비교 분석 |
| 3단계 | 3개월 | 멀티유저 권한 + 감사 로그 |
| 4단계 | 장기 | 온프레미스 AI 어댑터 |

## 【종합 에이전트 — 최종 개선 요건 보고서】

> **역할**: 4개 에이전트 주장 비판적 검토 및 실행 가능한 개선 로드맵 도출

### 1. 핵심 결론

mkt-desk는 딜러의 골든타임을 확보하는 실질적 가치가 있으나, **현재 상태로 운영하면 AI 환각·보안 위반 리스크가 편익을 상쇄한다.** 인간 승인 게이트와 정보보안 체계를 먼저 갖춘 뒤 단계적으로 고도화해야 한다. 속도보다 신뢰성이 선행되어야 시장 참여자가 실제로 의존하는 도구가 된다.

### 2. 즉시 개선이 필요한 사항 (Must-have)

**보안·컴플라이언스 — 최우선**
- 내부 포지션 데이터의 외부 AI API 전송 차단: Anthropic DPA 체결 또는 온프레미스 LLM(Ollama + Llama) 전환. 자본시장법 위반 리스크는 프로젝트 존폐 문제
- 외부 발송 이메일에 투자권유 Disclaimer 삽입, 내부 배포 전용으로 제한

**AI 환각 방지 — 인간 승인 게이트**
- 자동 발송 전 담당 딜러 1인의 검토·승인 의무화 (원클릭 승인 버튼)
- 기준금리·환율 등 핵심 수치는 공식 API 직접 조회 후 하드코딩으로 본문 삽입 → LLM이 수치를 생성하지 않도록 설계
- 데이터 타임스탬프 명시 + Fallback 알림 SOP 수립

### 3. 중기 개선 사항 (Should-have, 3개월 내)

- **수신자 분리 발송**: 경영진(한 줄 결론 버전) / 리스크관리부(이상 징후 강조 버전) / 영업점(시장 현황 요약 버전) 3종 템플릿
- **맥락화 단락 추가**: "왜 움직였나" — 트리거 이벤트, 트레이딩 임플리케이션, 리스크 시나리오 구조화 섹션 자동 생성
- **PostgreSQL 전환 + BullMQ 비동기 큐**: JSON 파일 한계 극복, 장애 격리 및 재처리 보장
- **이벤트 캘린더 연동**: 금통위·FOMC 일정 자동 감지 후 골든타임 리포트 우선 트리거

### 4. 장기 발전 방향 (Nice-to-have, 6개월+)

- **RAG 기반 과거 리포트 비교**: 유사 금리·환율 국면의 과거 리포트 검색, "이번과 다른 점" 자동 요약
- **딜러 체감 정보 입력 인터페이스**: 비공개 시장 정보를 딜러가 직접 입력하면 리포트에 반영되는 Human-in-the-loop 구조
- **JWT 권한 관리 + 감사 로그**: 승인자별 열람·발송 이력 추적, 내부 감사 대응
- **LLM 어댑터 패턴**: AI 제공사 전환(Anthropic → 온프레미스)에 대비한 인터페이스 추상화

### 5. 에이전트 간 쟁점 조율

| 쟁점 | 딜러 에이전트 | 리스크 에이전트 | 조율 결과 |
|---|---|---|---|
| 속도 vs. 안전성 | 빠른 배포 강조 | 승인 게이트 필수 | 안전성 우선 — 승인 게이트 도입해도 90분→15분 단축 충분히 달성 가능. 환각 사고 한 건이 신뢰를 영구 손상 |
| AI 한계 vs. 고도화 | (가치 에이전트) AI 체감 정보 대체 불가 | (기술 에이전트) RAG·예측 고도화 | AI는 "딜러 판단 대체"가 아닌 "딜러 시간 절약"으로 목표 재정의 |

### 6. 최종 권고

> **이 프로젝트의 진정한 가치는 "딜러가 생각할 시간을 버는 인프라"에 있다.**

리포트 자동화 자체가 목적이 아니라, 골든타임에 딜러가 수치 취합 대신 **판단과 의사결정에 집중**할 수 있도록 돕는 것이 본질이다.

지금 당장 해야 할 단 하나:

> **핵심 수치를 LLM이 아닌 공식 API로 직접 주입하는 "수치 하드코딩 파이프라인" 구축**

이것 하나만으로 환각 리스크의 80%를 제거하고, 조직 내 신뢰 확보의 기반이 마련된다.
