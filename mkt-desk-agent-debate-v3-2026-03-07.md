# mkt-desk 프로젝트 개선 토론 보고서 (3차)

**날짜**: 2026-03-07
**대상 프로젝트**: mkt-desk — 자금시장 리포트 자동 생성기
**참여 에이전트**: CRO/CFO / PB·영업팀 / 채권 애널리스트 / 퀀트·데이터 사이언티스트 / PM(조율)

## 【CRO/CFO 에이전트】

> **관점**: 최고리스크관리책임자 겸 CFO — 리포트 최종 의사결정 활용자

### 1. 경영진이 원하는 것: 판단 근거

수치는 Bloomberg가 이미 준다. 리포트에서 원하는 건 "지금 무엇을 해야 하는가"에 대한 근거. UST 10년이 5bp 올랐다는 사실이 아니라, 그것이 우리 조달비용·NIM·자본비율에 어떤 경로로 영향을 미치는지. 리포트는 반드시 **"시사점 → 행동 선택지 → 리스크 시나리오" 3단 구조**로 마무리되어야 한다.

### 2. 의사결정 연결을 위한 추가 요건

- 데이터 출처 명시, 모델 가정의 명시적 표기
- **전일 리포트 대비 판단 변화 이력** ("지난주 대비 스탠스 변경 이유"가 없으면 신뢰 불가)

### 3. AI 리포트 신뢰 구축 3조건

1. 사람이 검토·서명한 승인 이력
2. 과거 예측과 실제 결과의 트래킹 레코드
3. 오류 발생 시 책임 소재 명확화 ("AI가 썼다"는 면책은 경영 현장에서 통하지 않는다)

### 4. P0 요건에 대한 경영진 입장

DPA·감사 로그·승인 체계는 비용이 아니라 전제조건. 이것 없이 이사회에 AI 리포트를 올리는 순간 리스크 거버넌스 실패로 감독당국 지적 대상. ROI 계산 이전의 문제. P0 완비 이후 "자동화 절감 시간 × 인건비"로 **6개월 내 회수 가능**.

### 5. 경쟁력 기여 시나리오

딜러가 데이터 수집 대신 판단에 집중 → 경쟁 은행 대비 30분 빠른 내부 의사결정 → 조달 타이밍 실질 비용 우위. 장기적으로 이 리포트가 내부 리스크 모델의 입력값으로 진화하면 **AI 기반 동적 한도 관리** 가능.

> **결론**: 도구의 완성도보다 거버넌스의 완성도가 먼저. P0를 갖춘 뒤 "판단 근거를 제공하는 내러티브"로 진화할 때 비로소 경영 도구가 된다.

## 【PB·영업팀 에이전트】

> **관점**: WM센터 PB 팀장 + 기업금융 영업 경험자 — 고객 접점 최전선

### 1. 현재 방식의 문제

매일 아침 리서치 메일 복붙 + 인터넷 자료 PPT 붙여넣기. 새벽 금리·환율 급변 시 오전 미팅 전 업데이트 불가, 출처 불명확한 자료는 법인 CFO 앞에서 신뢰도 하락. mkt-desk의 ECOS·FRED 공식 데이터 기반이 이 두 문제를 동시 해결.

### 2. 고객군별 정보 니즈 차이

| 고객군 | 원하는 정보 |
|---|---|
| 개인 자산가 (HNWI) | "지금 달러 예금 들어야 하나요?" — 즉각적 의사결정 포인트 |
| 중견기업 CFO | 원/달러 방향성, 수출입 결제 타이밍, 단기 CP·CD 금리 추이 |
| 기관투자자 | 스프레드 변동, 듀레이션 리스크, 글로벌 통화정책 연계 분석 |

### 3. 계약 성사로 이어진 실제 사례

기준금리 인하 기대 국면에서 "현재 3.5% 정기예금이 6개월 후 2.8% 수준으로 하락 가능"이라는 구체 수치와 함께 장기 확정금리 상품 제안 → 당일 계약 성사. 리포트가 **"정보 전달"이 아닌 "Call to Action"** 으로 끝나야 한다.

### 4. 고객 대면 자료화 전제 조건

- 은행 리서치 브랜드 로고 + 법적 면책 문구 필수
- 모바일 카카오톡 공유 가능한 **1페이지 요약형** + PDF 풀버전 2종
- 언어 수준: 개인 고객 평어체 / 법인 전문 용어 분리

### 5. 시급히 추가되어야 할 기능 3가지

1. 고객 등급별 리포트 자동 분기 (HNWI / 법인 / 기관 템플릿)
2. 특정 지표 임계값 초과 시 PB 알림 + 초안 자동 생성 (예: 환율 1,450원 돌파)
3. 리포트 열람 여부 추적 + CRM 연동 (미열람 고객 → 전화 영업 우선순위)

> **결론**: 이 3가지가 갖춰질 때 mkt-desk는 단순 정보툴에서 **영업 생산성 도구**로 격상된다.

## 【채권 애널리스트 에이전트】

> **관점**: 국내 증권사 채권·금리 전략팀 수석 애널리스트 — 리포트 품질 평가자

### 1. 현재 데이터셋의 구조적 한계

| 부재 데이터 | 영향 |
|---|---|
| 외국인 국채 순매수·투신 듀레이션·Repo 잔고 | 수급 분석 공허 |
| KTBi(물가연동국채) | BEI 산출·실질금리 분리 불가 |
| CD91일·CP91일·콜금리 | 단기자금시장 스프레드 분석 불가 |
| NDF 누적 포지션·역외 스왑 플로우 | 환율-금리 상관관계가 현상 묘사에 그침 |
| Repo 시장 금리·담보 스트레스 지표 | 단기 유동성 리스크 포착 불가 |

### 2. AI가 잘하는 것 vs. 애널리스트 고유 영역

| AI 가능 | 애널리스트 고유 (대체 불가) |
|---|---|
| 멀티에셋 상관관계 패턴 서술 | 마이크로 수급 판단 (입찰 부진 원인 등) |
| 전일 대비 수치 변화 요약 | 비공개 IB 플로우 기반 방향성 추정 |
| 글로벌 매크로 데이터 정합성 점검 | 통화당국 커뮤니케이션 뉘앙스 해석 |
| | "현 레벨에서 매수/관망" 실질 투자 판단 |

### 3. 내러티브 품질 현주소

| 기준 | 수준 |
|---|---|
| Bloomberg 브리핑 대비 | 약 30% |
| KB증권 모닝 노트 대비 | 약 50% |

결정적 약점: "왜 지금 이 레벨인가"의 인과 논리 대신 "무엇이 움직였는가" 현상 나열에 머묾.

### 4. 반드시 포함해야 할 분석 프레임

- 수익률 곡선 기울기 (2s10s, 3s10s) + 경기 사이클 연동
- 실질금리 (명목 - BEI) 분리 분석
- 크레딧 사이클 (AA-/BBB- 스프레드 추세 + 발행시장 동향)
- IRS-국채 스왑 스프레드 (은행 수급 압력 진단)
- 글로벌 비교 (UST-KTB 스프레드, 한미 금리 역전 폭)

### 5. 애널리스트 수준 근접을 위한 개선 방향

- **즉시 데이터 추가**: CD91일, KTBi, 외국인 국채 보유잔고(ECOS), 기재부 국채 발행 캘린더
- **프롬프트 강제 4단 구조**: 전일 대비 변화 → 원인 가설 → 시장 함의 → 전략적 시사점
- **"현상 서술 금지"** 지침 명시
- **자기검증 루프**: 전주 전망 vs. 실제 결과 비교를 루틴에 내장 → 신뢰도 누적

## 【퀀트/데이터 사이언티스트 에이전트】

> **관점**: 자산운용사·증권사 퀀트 리서치팀 팀장 — 금융 데이터 분석·통계 모델링 전문

### 1. 당장 구현 가능한 정량 분석

기존 스냅샷 데이터에서 즉시 산출 가능한 파생 지표:

| 지표 | 산출 방법 | 의미 |
|---|---|---|
| 커브 기울기 | KTB10Y - KTB3Y | 경기 사이클 판별 |
| KTB-IRS 스왑 스프레드 | IRS5Y - KTB5Y | 시장 유동성·은행 수급 |
| BBB-AA 크레딧 스프레드 | corpBBBMinus3y - corpAAMinus3y | 신용 리스크 온/오프 |

### 2. 이상 탐지: 최적 복합 지수

**VIX + KoreaCDS5Y + corpSpread(BBB-AA) + 스왑포인트** 4개 동시 이탈 시 스트레스 경보. Z-score 기반 복합 지수, rolling 15일 평균/표준편차로 현재 히스토리 20영업일만으로도 즉시 구현 가능.

### 3. 방향성 예측: 실효적 간단 모델

과도한 ML보다 **회귀 기반 인터마켓 분석**이 현실적:
- `USD/KRW ~ (DXY + UST10Y + KOSPI + 스왑포인트)` 다중회귀
- `KTB10Y ~ (UST10Y + CPI 서프라이즈 + VIX)` 회귀

예측보다 **"잔차 모니터링"** 에 집중. 잔차 1.5σ 이탈 시 괴리 경보를 리포트에 반영.

### 4. AI 프롬프트 주입 통계 지표 6종

사전 계산 후 프롬프트에 구조화 텍스트로 주입:

| 지표 | 설명 |
|---|---|
| 10Y-3Y 스프레드 및 전일대비 변화 | 커브 방향 |
| KTB-IRS 스왑 스프레드 | 은행 수급 압력 |
| BBB-AA 크레딧 스프레드 방향 | 신용 위험 방향 |
| VIX 레벨 구간 | 15 미만 / 15~20 / 20 초과 |
| CDS-스왑포인트 괴리 여부 | 외환 스트레스 |
| Z-score 이탈 여부 | 이상 탐지 신호 |

### 5. HistoryPoint 구조 개선 방향

**현재 한계**:
- `{ date: string, value: number }` 단일 시계열 → 크로스 지표 상관 분석 불가
- 20영업일 → 변동성 추정 신뢰도 낮음
- 시간대 정보 부재 → 장중 데이터 확장 불가

**개선안**:
```typescript
// 현재
interface HistoryPoint {
  date: string
  value: number
}

// 개선
interface MultiTimeSeriesPoint {
  date: string
  values: Record<SeriesKey, number>
  z5d: number   // 5일 Z-score
  z20d: number  // 20일 Z-score
}
// + 60영업일 이상 데이터 확보 권장
```

### PM 최종 권고 — 3차 토론을 통해 가장 크게 바뀐 시각

> **"이 시스템의 핵심 산출물은 리포트가 아니라 판단 이력이다."**

1·2차까지 논의의 중심은 리포트를 얼마나 정확하고 안전하게 만드느냐였다. 그러나 3차에서 채권 애널리스트·퀀트·CRO 세 에이전트가 공통으로 가리킨 방향은 다르다.

전주 전망 vs. 실제 비교, 스탠스 변경 이력, 과거 예측 트래킹 — 이 세 가지는 모두 **"AI가 어떤 판단을 했고, 그 판단이 맞았는가"를 추적하는 메타 레이어**다.

리포트 한 장 한 장보다, 수십 회 판단의 누적 정확도가 이 시스템의 신뢰 자산이 된다. P2의 자기검증 루프는 단순 기능 개선이 아니라 **AI 시스템이 금융기관 내부에서 장기 신뢰를 획득하는 유일한 경로**다.

→ **HistoryPoint 구조 개선 + 60영업일 이력 확보를 P2 최우선 항목으로 격상 권고.**
